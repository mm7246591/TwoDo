# UI 審查清單

在進行大幅修改前，以及完成前的最後驗收時，都可以使用這份清單。

## 層級

- 在一個畫面內就讓頁面目的清楚可見。
- 在三秒內讓主要操作清楚可辨。
- 確保標題、輔助文字與中介資訊有明確不同的視覺強度。
- 移除會和關鍵任務內容競爭注意力的裝飾元素。

## 版面

- 讓不同區塊之間的間距節奏保持一致。
- 如果巢狀容器沒有增加意義，就減少它們。
- 每個畫面只採用一種背景策略：純色、分層，或漸層主導。
- 讓卡片群組與表單群組看起來是有意識對齊的。

## 元件

- 在卡片、輸入框與按鈕之間重用同一套圓角系統與陰影系統。
- 讓次要操作在視覺上比主要操作更低調。
- 讓空狀態、載入狀態與錯誤狀態延續整個頁面的視覺語言。
- 讓圖示扮演輔助角色，而不是搶走主視覺。

## 表單

- 每個欄位都要搭配清楚標籤，或至少有不會被誤解的上下文。
- 在行動裝置上維持舒適的欄位內距。
- 讓 focus 狀態清楚可見，但不要看起來像壞掉。
- 錯誤提示不要只靠顏色傳達。

## 行動裝置

- 點擊區至少維持 44 到 48px 的寬或高。
- 留意安全區、固定操作列，以及鍵盤遮擋問題。
- 在小螢幕上避免過於擁擠的雙欄版面。
- 檢查長文字、電子郵件地址與翻譯後字串是否能自然換行。

## 無障礙

- 在可行情況下，小字對比至少維持 4.5:1。
- 在可行情況下，大字與關鍵 UI 對比至少維持 3:1。
- 不要只靠顏色來傳達成功、警告或錯誤。
- 動態效果應有幫助、簡短，並對敏感使用者足夠友善。

## 最後檢查

- 每個畫面最多只保留一個主強調色，以及一組輔助亮點色系。
- 每個畫面最多只保留一到兩個高戲劇性的重點表面。
- 確保即使拿掉所有裝飾性修飾，介面仍然保持一致。
- 讓使用者看到的是比原本更清楚、更平靜，也更值得信任的畫面。
