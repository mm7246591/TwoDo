# 風格方向

當使用者要求重新設計、希望更主流的風格，或想讓產品質感更精緻時，使用這份文件。

## 1. 俐落 SaaS 風格

適用於偏重生產力、類儀表板，或高度需要建立信任感的產品。

- 視覺特徵：冷色或中性色基底、單一強調色、俐落字體、寬鬆卡片、細緻邊框、低噪音背景、克制陰影。
- 適合：任務工具、登入註冊流程、後台區域、設定頁、筆記 App、偏輕量的 B2B 產品。
- 避免：過度活潑、太多漸層、多個強調色、戲劇化動態。

## 2. 溫潤生產力風格

適用於需要讓產品顯得有幫助、友善，並且讓人願意天天回來使用的情境。

- 視覺特徵：柔和中性色、較溫暖的亮點、圓潤表面、較輕柔的陰影、支持性的文案語氣、較平順的對比變化。
- 適合：待辦 App、習慣追蹤、健康工具、學習產品、社群型工具。
- 避免：到處都是幼稚插圖、糖果色過量、文字對比過淡。

## 3. 鮮明編輯科技風格

適用於需要更鮮明品味與更強視覺自信，但又不希望變得太小眾的產品。

- 視覺特徵：更明顯的字級層級、更大留白、較少但更有力的區塊、俐落節奏、節制使用強調色、刻意安排的不對稱。
- 適合：落地頁、功能介紹頁、創作者工具、高級感生產力產品。
- 避免：把資訊密集的 App 畫面做成海報式版面，或每個區塊都套用超大的 hero 視覺。

## 4. 柔和表現層次風格

適用於使用者希望畫面更有當代感與活力，但仍需保持良好可用性時。

- 視覺特徵：更突出的標題、容易掃視的 chips 與按鈕、柔和動態、較有變化的色彩強調、節制使用模糊或分層景深，以及一到兩個重點表面。
- 適合：現代行動 App、導覽 onboarding、有重點摘要的儀表板、高辨識度的入口畫面。
- 避免：全螢幕發光、厚重玻璃擬態、拖慢任務完成速度的裝飾性動態。

## 選擇規則

- 複雜或工具導向的介面，預設使用 `俐落 SaaS 風格`。
- 待辦、習慣、個人化與生活型工具，預設使用 `溫潤生產力風格`。
- 若頁面以行銷表達為主，或需要更強品牌存在感，使用 `鮮明編輯科技風格`。
- 只有在使用者明確希望更有個性，且畫面能承受額外強調而不損失速度與清晰度時，才使用 `柔和表現層次風格`。

## 當前趨勢綜合

以下趨勢在當代產品 UI 中普遍具有高接受度：

- 在不破壞結構的前提下，讓字體、重點表達與動態更有個性
- 透過節制使用的陰影、模糊與分層，建立柔和層次
- 透過強調色與一眼可辨識的亮點，增加個人化感受
- 採用更溫暖、更有人味的表面，而不是完全冰冷的企業風格
- 把無障礙與行動裝置舒適度當成基線，而不是事後補救

對大多數應用畫面來說，版面應保持沉穩，只在焦點區域使用帶有趨勢感的視覺亮點。
