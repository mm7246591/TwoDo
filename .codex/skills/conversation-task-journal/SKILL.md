# Conversation Task Journal

用「對話命名 + 任務檔 + 進度檔」的方式，把每次對話要做什麼、做到哪裡、下一步是什麼外部化，避免對話被摘要/壓縮後遺漏關鍵上下文。

## 核心規則

- 將「任務檔」視為本次對話的需求與規劃（scope/清單/驗證方式）來源。
- 將「進度檔」視為本次對話的執行紀錄與決策紀錄（timeline）。
- 同一個對話期間，檔名與位置要固定；不要每回合重命名或分散到不同資料夾。
- 若專案另有全域任務文件（例如 `任務.md` / `任務進度.md`）的必讀規範，仍需先讀取並遵守；本 skill 的對話檔不取代專案全域規劃。

## 工作流程

### 1) 初始化（本次對話第一次使用時）

1. 決定「本次對話名稱」：
   - 優先使用使用者提供的名稱；若沒有，使用 `YYYY-MM-DD-主題-kebab-case`（只用英數與 `-`，避免空白）。
2. 決定檔案路徑（固定放 `.mission/`）：
   - `.mission/{對話名}-任務.md`
   - `.mission/{對話名}-任務進度.md`
3. 確保 `.mission/` 資料夾存在（若不存在先建立）。
4. 建立/初始化兩個檔案：
   - 任務檔：寫入本次對話的目標、範圍、任務拆分與驗證方式。
   - 進度檔：寫入「目前狀態 = 未開始」，並保留後續每次更新的區塊。
5. 建立「當前對話指標檔」以便後續回合穩定續寫（避免忘記對話名）：
   - 建議路徑：`.mission/conversation-task-journal-current.md`
   - 內容至少包含：對話名、任務檔路徑、進度檔路徑、對話起始時間、對話主題摘要。

### 2) 每次執行任務前（任何實作/改檔/跑指令之前）

1. 讀取 `.mission/conversation-task-journal-current.md`，取得本次對話檔案位置。
2. 讀取並比對任務檔與進度檔（以「當前對話指標檔」內紀錄的檔案路徑為準）：
   - 任務是否已變更/追加？若需求有變，先更新任務檔並在進度檔記錄「變更原因與影響」。
   - 哪些任務已完成/進行中/卡住？以任務檔的清單為主，並同步勾選/標記狀態。
3. 產出「本回合要做的最小下一步」並開始執行。

### 3) 本回合任務完成後（回覆使用者前）

1. 更新進度檔（以「當前對話指標檔」內紀錄的檔案路徑為準）：
   - 本回合完成了什麼（對應任務 ID/條目）
   - 變更了哪些檔案/關鍵行為（列路徑即可）
   - 還剩什麼、下一步是什麼、是否有風險/待確認
2. 視需要更新任務檔（以「當前對話指標檔」內紀錄的檔案路徑為準）：
   - 勾選已完成的任務
   - 若 scope 變更，更新「範圍/成功定義/驗證方式」並同步到進度檔

## 檔案範本

### `.mission/{對話名}-任務.md`

```md
# {對話名} 任務

## 目標
- ...

## 範圍
- 包含：
- 不包含：

## 任務清單
- [ ] T1：...
- [ ] T2：...
- [ ] T3：...

## 驗證方式
- ...
```

### `.mission/{對話名}-任務進度.md`

```md
# {對話名} 任務進度

## 狀態摘要
- 最後更新：YYYY-MM-DD HH:mm
- 目前狀態：未開始 / 進行中 / 已完成 / 阻塞
- 下一步：...

## 變更紀錄
### YYYY-MM-DD HH:mm
- 完成：
- 調整：
- 影響：
- 下一步：
```
