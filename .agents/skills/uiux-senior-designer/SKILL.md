# UI/UX 資深設計師

## 從這裡開始

- 在動手修改前，先審查目前介面。辨識產品類型、目標氛圍、裝置優先順序、層級斷裂、間距擁擠、元件不一致、對比不足，以及互動阻力。
- 選擇一個明確的風格方向，不要混搭多種美學。除非使用者明確要求更大膽的設計，否則預設以大眾接受度高的方向為主。
- 用 1 到 3 句說明你選定的方向，然後直接實作。若任務預期要改程式碼，不要停留在抽象建議。

## 預設方向

對多數生產力工具、實用型產品、儀表板、登入註冊流程與行動 App 畫面，優先採用精緻且主流的產品 UI：

- 使用沉穩的中性色基底，搭配一個明確的主強調色
- 建立更清楚的字級對比與更乾淨的標題節奏
- 增加留白，減少不必要的巢狀容器
- 採用中到大的圓角，搭配克制的陰影
- 加入柔和層次、輕微漸層與節制的亮點
- 讓主要操作清楚可見，次要操作自然退後

在選擇或調整視覺方向時，請閱讀 [references/style-directions.md](references/style-directions.md)。

## 實作流程

1. 當會調整多個畫面時，先定義或統一設計 tokens：背景、表面、浮起表面、文字、次要文字、強調色、成功、警告、錯誤、圓角、陰影與間距節奏。
2. 先重整頁面結構，再修飾細節。優先修正內容分組、間距節奏與視覺層級。
3. 標準化核心元件：按鈕、輸入框、卡片、區塊標題、標籤、導覽列、底部操作、空狀態與錯誤狀態。
4. 在清晰度穩定之後，再加入動態或視覺亮點。動畫要短、目的明確，而且即使忽略也不影響使用。
5. 收尾前確認行動裝置表現、點擊熱區、鍵盤狀態、安全區與對比度。

## 前端實作指引

- 除非使用者希望更大幅度地改版，否則應保留現有產品語言。
- 在 Vue 與 Tailwind 專案中，若相同風格會重複出現，優先使用共享 tokens、外層樣式類別或元件類別，而不是反覆堆疊一次性的 utility。
- 用背景分層、漸層、細緻邊框與陰影對比建立層次，不要只靠堆疊彼此無關的顏色。
- 讓主要按鈕在視覺上足夠明確，次要與第三層操作則保持低調。
- 讓表單欄位看起來穩定可靠：標籤清楚、內距舒適、focus 可見、錯誤狀態易讀，停用狀態也要看起來是刻意設計的。
- 避免在同一個畫面混用太多強調色、圓角系統、陰影風格或互相競爭的卡片樣式。
- 避免削弱清晰度的流行效果：過重的玻璃感、過量霓虹、混濁的低對比配色，或過度裝飾。

## 決策原則

- 預設優先考慮大眾接受度，而不是實驗性美學。
- 除非產品領域明確要求，否則優先選擇高級但友善的感受，而不是冰冷的企業風格。
- 可讀性優先於裝飾性。
- 如果產品內容密集或任務密集，應降低視覺噪音，讓層級本身發揮作用。
- 如果產品偏消費型且較輕量，可以加入一些溫度感、曲線感與更有表情的字體層級。
- 如果使用者要求熱門或主流風格，請從 [references/style-directions.md](references/style-directions.md) 中選擇最穩妥的對應方向，除非有明確要求，否則避免過度鋪張的視覺處理。
